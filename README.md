# 个人记账软件

一个功能完整的个人记账 Web 应用，支持二级分类、报销管理、退款管理、预算管理、数据统计和跨设备同步。

## 功能特性

### 📝 记账
- **收支记录**：支持收入和支出两种类型
- **二级分类**：大类下细分（如餐饮 → 早餐/午餐/晚餐），支持自定义
- **支付方式**：微信支付、支付宝、现金、银行卡、信用卡
- **精确时间**：记录到分钟
- **备注**：支持自定义备注

### 📊 账单与统计
- **月度账单**：按月查看，收支汇总
- **统计图表**：饼图（支出分布）、折线图（收支趋势）、柱状图（每日支出）
- **分类排行**：各分类支出排名
- **报销统计**：待报销/已报销统计

### 💰 预算管理
- 按月设置每个支出分类的预算上限
- 总预算概览
- 进度条实时显示，超 80% 警告，超 100% 红色提示

### 🧾 报销功能
- 记账时标记"可报销"，选择报销账户（出差、垫付、团建等）
- 报销账户支持自定义添加
- 在"报销管理"中查看所有待报销记录
- 标记已报销后，该笔支出不再计入统计和预算

### ↩️ 退款功能
- 点击账单可直接标记退款
- 默认全额退款，支持修改金额
- 自动生成退款收入记录

### ☁️ 跨设备同步
- **GitHub Gist 自动同步**：配置 Token 后每次记账自动推送到云端
- **扫码同步**：设备间生成/扫描二维码，无需账号

### 📁 数据管理
- 导出 JSON / CSV
- 导入 JSON（覆盖或合并）
- 数据概览（总交易数、数据范围、存储大小）

## 快速开始

### 本地使用
直接用浏览器打开 `index.html` 即可，无需服务器。

### 部署到 GitHub Pages

1. Fork 或创建仓库
2. 将 `index.html` 和 `README.md` 上传到仓库
3. 仓库 Settings → Pages → 选择 main 分支 → 保存
4. 访问 `https://你的用户名.github.io/仓库名/`

### 多设备同步配置

1. 打开 **我的 → 同步与备份**
2. 配置 GitHub 个人访问令牌（Settings → Tokens → 勾选 `gist` 权限）
3. 开启**自动同步**
4. 点**推送数据**，在其他设备上点**拉取数据**

## 技术说明

- 纯前端应用，单 HTML 文件
- 数据存储在浏览器 localStorage 中
- Chart.js 用于图表展示
- GitHub Gist API 用于云同步
- 无需后端服务器

## 数据安全

- 所有数据存储在浏览器本地
- 更新 HTML 文件不会丢失已有数据
- 建议定期到"数据管理"导出备份
- Gist 同步数据存在 GitHub 私有 Gist 中
