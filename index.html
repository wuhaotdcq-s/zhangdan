个人记账软件 v1.2.0

数据存储在浏览器本地 (localStorage)，与程序代码分离

GitHub Pages 部署
  1. 在 GitHub 创建仓库
  2. 将本 HTML 文件上传（重命名为 index.html 更方便）
  3. 仓库 Settings → Pages → 选择 main 分支 → 保存
  4. 访问 https://你的用户名.github.io/仓库名/
GitHub 仓库同步配置
  1. 去 GitHub Tokens → Generate new token (classic)
  2. 勾选 repo 权限，生成令牌
  3. 在"我的 → 同步与备份"中填入 Token 和仓库地址（格式：用户名/仓库名）
  4. 点击"同步到仓库"，数据将保存为仓库中的 ledger-data.json
  5. 另一设备配相同 Token 和仓库地址，点"从仓库拉取"即可同步
扫码同步（无需账号）
  1. 设备A：同步与备份 → 生成同步码
  2. 设备B：同步与备份 → 扫描同步码
  3. 适合少量数据快速传输
报销功能
  • 记一笔时打开"可报销"开关，标记待报销支出
  • 在"我的 → 报销管理"中查看所有待报销记录
  • 报销后标记为"已报销"，账单中会显示状态
分类与二级分类
  • 选择一级分类（如餐饮）后自动显示子分类
  • 子分类在"我的 → 分类管理"中添加/编辑
  • 用逗号分隔多个子分类
数据安全
  • 所有数据存储在浏览器 localStorage 中
  • 更新 HTML 文件不会丢失数据
  • 建议定期在数据管理中导出备份
  • 仓库同步数据存在 GitHub 仓库的 ledger-data.json 文件中
